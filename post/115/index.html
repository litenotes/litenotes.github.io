Litespeed安装

Posted on 2020-10-27，3 min read

LiteSpeed的安装比OpenLiteSpeed难度大，网上教程太少，官方文档非常晦涩难懂，因为是商业软件，免费版也需要获取序列号，这就会面临一个坎，安装的时候，也会遇到无法运行的情况，这就是第二个坎。

LiteSpeed序列号

由于是商业版本，需要正版序列号才能运行，不过，对于小站来说，还是福利，如果你的服务器配置低于要求，是可以免费使用的。

如果是一个域名、1个worker、低于2G内存，就可以使用Free Starter，但是还是需要订购序列号。点击Buy Now就可以购买：

点击continue，再点击checkout，填入详细信息（在这里注册邮箱），即可得到序列号，序列号会发送至注册的邮箱，同时还会发送一份简单的安装指南。
获取序列号是第一个坎，因为一直会卡在真人验证阶段，而且似乎没有任何解决办法。其实是谷歌的recaptcha无法连接而已，推荐的解决办法参考我之前的文章：谷歌recaptcha无法连接简单解决方案

LIteSpeed的安装

LLStack 一键安装包

如果想简单就安装使用LiteSpeed，推荐使用LLStack一键安装包，提供完整的教程、各环节均有详细操作说明，推荐使用。

LLStack 全称是 “Linux LiteSpeed Stack”，即在 Linux 上安装 LiteSpeed + PHP + MySQL/MariaDB（可选）的高性能 Web 运行环境，特别适合运行 PHP 程序。
LLStack 仅适用于 RHEL 7 系操作系统及其衍生版，RHEL7、CentOS7、OracleLinux7、CloudLinux 7等。 RHEL 8 系正在测试中。
https://www.llstack.com/

官方安装包

因为LiteSpeed已经支持CentOS8，因此我选择手动安装。于是，困难就出现了，官方的安装包安装后，无法进入Web控制面版，提示503错误。通过查询官方文档、论坛等等均无解决办法。获取序列号后按邮件的指导一键代码安装依然无法正常运行。
RPM安装
官方提供RPM安装途径：Install LiteSpeed Web Server from RPM，我是通过这种方式安装成功。
①添加库
CentOS 7:

rpm -ivh http://rpms.litespeedtech.com/centos/litespeed-repo-1.1-1.el7.noarch.rpm

CentOS 8：

rpms.litespeedtech.com/centos/litespeed-repo-1.2-1.el8.noarch.rpm

其他版本的库可以到这里查看：http://rpms.litespeedtech.com/centos/
②安装LSWS包
yum install lsws
这是官方提供的全自动程序，安装成功后，打开主机IP地址的7080端口（防火墙等需要放行），就可以看到Web后台，这个后台运行速度比OpenLiteSpeed更快。
